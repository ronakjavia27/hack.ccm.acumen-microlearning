            AI-Powered Clinical Microlearning Platform
================================================================

QUICK START
-----------
1. Copy .env.example to .env and fill in your API keys
2. Place PDF files inside input_pdfs/articles/ or input_pdfs/guidelines/
3. Run:    python generator.py
   This watches the input folder and processes PDFs one by one

THE THREE SCRIPTS
-----------------
  generator.py    Ingests PDFs, creates summaries + pearls (daily use)
  maintainer.py   Checks & fixes the output files (weekly maintenance)
  syncer.py       Syncs to git, sends emails (periodic sync)

================================================================
1. GENERATOR.PY - PDF INGESTION PIPELINE
================================================================

WHAT IT DOES:
  Takes PDF files from input_pdfs/ and produces:
  - Structured JSON summaries (Pass 1 - uses OpenRouter with deepseek-ai/DeepSeek-V4-Pro by default)
  - Clinical pearls / key takeaways (Pass 2 - uses Together AI)
  
  All output goes into output_files/ organized by specialty and type.

MODES:
  --mode watch         (DEFAULT) Watch folder, process each PDF fully
                       Runs Pass 1 (summary) + Pass 2 (pearls) per PDF
  
  --mode summary       Only run Pass 1 (summaries). Skip pearls.
                       Use when you only want structured summaries.
  
  --mode pearls        Only run Pass 2 on files that are missing pearls.
                       Use to fill in pearls for already-summarized files.
  
  --mode summary_pearls
                       Phase 1: Run Pass 1 on ANY pending PDF
                       Phase 2: Run Pass 2 on ALL files missing pearls
                       Use for a complete run from scratch.

LLM PROVIDER (for Pass 1 extraction):
  --llm openrouter      Use OpenRouter (DEFAULT, reads OPENROUTER_API_KEY from .env)
  --llm together        Use Together AI (requires TOGETHER_API_KEY)
  --llm gemini          Use Google Gemini (requires PRIMARY_GEMINI_API_KEY)
  --llm other           Custom OpenAI-compatible API (also set --api-key and --model)

COMMON COMMANDS:
  python generator.py                              # Watch mode (default)
  python generator.py --mode summary               # Summaries only
  python generator.py --mode pearls                # Pearls only for pending
  python generator.py --mode summary_pearls        # Both, sequential
  python generator.py --status                     # Quick dashboard
  python generator.py --once                       # Process once, then exit
  python generator.py --dry-run                    # Preview only, no AI calls
  python generator.py --max 10                     # Process at most 10 files
  python generator.py --ocr                        # Enable OCR for scanned PDFs
  python generator.py --verbose                    # Show detailed per-file output
  python generator.py --reprocess path/to/pdf      # Force re-process one PDF
  python generator.py --extract-pearls path.json   # Run pearls only on one JSON
  python generator.py --llm together                # Use Together AI instead of OpenRouter
  python generator.py --llm gemini                  # Use Gemini instead
  python generator.py --llm openrouter --model anthropic/claude-3.5-sonnet  # Override OpenRouter model

EXAMPLES:
  # Quick status check:
  python generator.py --status

  # Process 5 new files with summaries only:
  python generator.py --mode summary --max 5

  # Add pearls to files that are missing them:
  python generator.py --mode pearls

  # Full pipeline from scratch:
  python generator.py --mode summary_pearls

  # Test run - see what would happen without making changes:
  python generator.py --dry-run

  # Use Gemini instead of OpenRouter for extraction:
  python generator.py --llm gemini

  # Use Together AI (original default):
  python generator.py --llm together

================================================================
2. MAINTAINER.PY - REPOSITORY HEALTH & REPAIRS
================================================================

WHAT IT DOES:
  - Ensures all output JSON files are valid
  - Removes duplicates from sent_summaries.json
  - Re-extracts pearls for files that failed
  - Checks for schema violations and fixes them
  - Generates health_report.json + health_report.md
  - Reads master_error_list YYYY-MM.txt to prioritize fixes

MODES:
  (no args)    Generate full health report (JSON + Markdown)
  --reconcile         Sync disk files with sent_summaries.json
  --validate          Schema-check all JSON output files
  --repair            Auto-fix safe issues (DOIs, specialties, etc.)
  --reclassify        Fix pearl system assignments from summaries
  --reprocess-pearls  Re-run pearl extraction for failed files
  --error-priority    Show prioritized action plan from error logs
  --auto-fix          Run EVERYTHING: reconcile + validate + repair + reprocess
  --full-scan         Include ALL monthly error logs (not just current month)
  --report-only       Generate health report without any fixes
  --reset-pearls FILE Remove and re-extract pearls for one JSON file
  --since YYYY-MM-DD  Filter error/report analysis to recent entries only
  --dry-run           Preview only, no changes
  --verbose           Show detailed per-file output

COMMON COMMANDS:
  python maintainer.py                              # Health report (default)
  python maintainer.py --repair                     # Fix safe issues
  python maintainer.py --error-priority             # See what needs attention
  python maintainer.py --auto-fix                   # Fix everything
  python maintainer.py --reprocess-pearls           # Re-run failed pearl extractions
  python maintainer.py --reset-pearls output_files/Cardiology/Review/paper.json
  python maintainer.py --error-priority --since 2026-06-01
  python maintainer.py --dry-run --auto-fix         # Preview auto-fix without changes

================================================================
3. SYNCER.PY - CLOUD SYNC & EMAIL DISPATCH
================================================================

WHAT IT DOES:
  - Syncs output files to git (with automatic backup tags)
  - Sends email digests to subscribers
  - Syncs subscriber list from Google Forms/Sheets

You run 
		python -m acumen_core.subtopic_mapper ‚Üí interactive prompt to assign proper subtopic
Or 
		python -m acumen_core.subtopic_mapper --batch-llm ‚Üí auto-classify via LLM
Then 
		python maintainer.py --subtopics ‚Üí syncs to both JSON files

MODES:
  (no args)    Same as --mode all (git sync)
  --mode all           Git add -A, commit, push (DEFAULT)
  --mode data          Git all files EXCEPT main_app.py
  --mode web           Git ONLY main_app.py
  --mode pearls        Git ONLY pearls.json + sent_summaries.json
  --mode email         Send pending emails to subscribers
  --mode subscribers   Sync Google Sheets -> emails.csv
  --mode full          subscribers -> email -> git all (sequential)
  --verify             Pre-flight health check before syncing
  --dry-run            Preview only
  --verbose            Detailed logging

COMMON COMMANDS:
  python syncer.py                              # Git sync (default)
  python syncer.py --mode all --verify          # Check + git sync
  python syncer.py --mode data                  # Sync data files only
  python syncer.py --mode pearls                # Sync pearls only
  python syncer.py --mode email                 # Send pending emails
  python syncer.py --mode subscribers           # Sync subscriber list
  python syncer.py --mode full                  # Everything sequentially
  python syncer.py --dry-run                    # Preview without changes

NOTE: Each git push creates a local backup tag named backup-YYYYMMDD-HHMMSS
      so you can always recover the previous state.

=============================================================
ACUMEN =  DASHBOARD
============================================================

WHAT IT IS:
  A FastAPI web dashboard and admin console for managing the
  hack.CCM content repository. Mounted at /console alongside
  the existing FastAPI app (main_app.py).

MODULES (content types):
  - Summaries   (sent_summaries.json ledger + output_files/*.json)
  - Pearls      (pearls.json ‚ÄĒ 1991 clinical pearls)
  - Theory      (theory.json ‚ÄĒ guideline documents with blocks)

DASHBOARD:
  Open http://localhost:8878/console in a browser.
  Browse, search, edit, and push content through a web UI.

  Features:
  - Table view for each module with text search + specialty/subtopic filters
  - Click any row to open the full editor with toolbar (bold, italic, lists)
  - Change a summary's System/Subtopic ‚Üí cascade preview shows affected
    pearls ‚Üí confirm to bulk-update all pearls for that file
  - Per-file and bulk git push with live status polling
  - Backup / restore (working tree + git bundle)
  - Audit log of all edits
  - Orphan scan: find files on disk not tracked in the ledger
  - Failed scan: flag items that failed pearl extraction

API ENDPOINTS (all under /console/api):
  GET  /info                         Server info
  GET  /modules                      Module specs (name, label, fields)
  GET  /{kind}                       List items (?q=&system=&status=&type=)
  GET  /{kind}/{id}                  Single item detail
  PUT  /{kind}/{id}                  Update item fields
  POST /push                         Push one file to git
  POST /push-all                     Push multiple files
  GET  /push/{job_id}                Poll push status
  GET  /backup                       List backups
  POST /backup                       Create backup (?label=...)
  POST /backup/restore               Restore from backup
  GET  /backup/verify/{name}         Verify backup integrity
  GET  /audit                        Tail audit log (?n=200)

CONSOLE ENVIRONMENT:
  PYTHONPATH is set automatically; just run from repo root.

  Quick start (from repo root):
    pip install -r requirements.txt
    uvicorn dashboard_app:app --host 0.0.0.0 --port 8878

  Or:
    python dashboard_app.py

  Then open: http://localhost:8878/console
  (Or http://localhost:8878 will redirect there)

  Go to http://localhost:8878/console

BACKUP CLI (standalone):
  python ccm_backup.py backup --label "before-big-edit"
  python ccm_backup.py list
  python ccm_backup.py verify <name>
  python ccm_backup.py restore --from-backup <name> --to <target-dir>

  Note: restore needs --force to overwrite existing target.
